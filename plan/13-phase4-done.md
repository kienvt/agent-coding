# T13 — Phase 4: Done

> **Phụ thuộc:** T09
> **Output:** `src/orchestrator/phase4-done.ts`

---

## Mục tiêu

MR đã được approve. Merge vào main, đóng tất cả issues, cleanup branches, post final report.

---

## Các bước

### Bước 1: Trigger Phase 4

Từ orchestrator khi nhận `MR_REVIEW` với action `approved` hoặc `MR_MERGED` event:
```
stateManager.transitionPhase → 'MERGING'
phase4Done.run(projectId, repo)
```

### Bước 2: `phase4Done.run(projectId, repo)`

Invoke agent với prompt:

```
The MR has been approved. Execute the Done phase.

GitLab project ID: {gitlab_project_id}
MR IID: {mrIid}
Issues to close: {iids.join(', ')}
Working directory: {repo.local_path}

STEP 1 - MERGE MR:
  glab mr merge {mrIid} --squash=false --delete-source-branch

STEP 2 - VERIFY MERGE:
  glab mr view {mrIid} --output json
  Confirm state === 'merged'

STEP 3 - CLOSE ALL ISSUES:
  For each issue IID in [{iids}]:
    glab issue close {iid}
    glab issue note {iid} --message "✅ Closed — implemented and merged in MR !{mrIid}"

STEP 4 - CLEANUP LOCAL BRANCHES:
  git fetch --prune
  Delete local feature branches that have been merged

STEP 5 - GENERATE FINAL REPORT:
  Create a summary with:
  - Total issues completed
  - MR URL
  - Date completed
  - Brief summary of what was implemented

STEP 6 - POST REPORT:
  glab mr note {mrIid} --message "{final_report}"

STEP 7 - Output on final line:
  "PHASE_COMPLETE: done"
```

### Bước 3: Sau khi agent xong

- Parse "PHASE_COMPLETE: done" từ agent output
- `stateManager.transitionPhase → 'COMPLETE'`
- Log completion metrics

### Bước 4: Xử lý merge conflict (nếu merge fail)

Nếu `glab mr merge` fail do conflict, agent nhận thêm instruction:
```
Merge failed due to conflicts. Resolve them:
  git fetch origin
  git checkout {sourceBranch}
  git merge origin/main
  # Resolve conflicts in conflicted files
  git add .
  git commit -m "chore: resolve merge conflicts"
  git push origin {sourceBranch}
Then retry: glab mr merge {mrIid}
```

- [ ] Tất cả issues được close với `glab issue close`
- [ ] Closing comment được post trên mỗi issue
- [ ] Feature branches được cleanup (local)
- [ ] Final report được post lên MR comment
- [ ] State transition cuối: `MERGING` → `COMPLETE`
- [ ] Nếu merge conflict → agent tự resolve và retry
